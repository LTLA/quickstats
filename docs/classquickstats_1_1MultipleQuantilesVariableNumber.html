Detailed Description

template<typename Output_, typename Number_, class QuantilesPointer_>
class quickstats::MultipleQuantilesVariableNumber< Output_, Number_, QuantilesPointer_ >

Calculate multiple quantiles from a variable number of elements.

Template Parameters
Output_Floating-point type of the output quantile. This should be capable of representing NaNs.
Number_Integer type of the number of elements.
QuantilesPointer_Pointer to a container that has a size() method and supports access by []. This may or may not be a smart pointer.

This uses the same logic as the MultipleQuantilesFixedNumber class but supports any number of elements, from zero up to a specified maximum.

Overload to compute the desired quantile from a sparse vector of length num. This vector is assumed to have nnz structural non-zeros and num - nnz zeros.

This method is not thread-safe.

Template Parameters
Input_Numeric type of the input values.
OutputFun_Functor that accepts a Number_ and an Output_.
Parameters
numNumber of elements from which to compute the quantile. This should be no greater than max_num.
nnzNumber of elements that are structural non-zero elements. This should be no greater than num.
[in]ptrPointer to the start of an array of length num. This is expected to contain the values of the structural non-zero elements of the sparse vector. It should not contain any NaN values. On output, the elements may be reordered.
outputFunction that accepts a Number_, the index of the probability in quantiles; and an Output_, the computed value of the quantile. This will be called once for each quantile, in order of increasing index from 0 to quantiles.size() - 1.
